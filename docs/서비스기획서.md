# 서비스 기획서 — 🧵 타래 (Tarae)

## 1. 서비스 개요

| 항목 | 내용 |
|---|---|
| 서비스명 | 타래 (Tarae) |
| 한 줄 정의 | 스쳐 지나가는 생각을 가볍게 기록하고, 서로 다른 생각을 엮어 새로운 아이디어로 발전시키는 개인 생각 관리 서비스 |
| 목적 | 메모 앱처럼 생각을 저장만 하는 데서 그치지 않고, 시간이 지나 잊혀진 생각들을 다시 연결해 실제 실행 가능한 기획으로 이어지도록 돕는다 |
| 타겟 사용자 | 사이드 프로젝트·콘텐츠 기획 등 여러 개의 아이디어를 동시에 굴리는 1인 개발자/기획자, 순간적인 생각이 자주 휘발되어 아쉬움을 느끼는 사용자 |
| 배포 URL | https://think-tarae.vercel.app/ |
| GitHub | https://github.com/woniri/codyssey-A1-3 |

## 2. 페이지/섹션 구성

내비게이션 바를 통해 4개 화면을 자유롭게 이동한다.

1. **실마리 (홈, `index.html`)**
   - 생각을 한 줄로 던지는 입력창
   - 오늘의 인사이트(최근 생각 경향 요약), 태그 클라우드, 실타래 상태(풀려 있는 실가닥/단단해진 타래/먼지 쌓인 실가닥) 지표
2. **베틀 (`loom.html`)**
   - 저장된 생각 목록에서 하나를 골라 4가지 관점으로 확장해보는 작업대
3. **타래장 (`vault.html`)**
   - 전체 생각/프로젝트를 네트워크 그래프로 시각화
   - 두 개의 생각 노드를 선택하면 AI가 조합하는 **실타래 연금술** 기능 제공 (핵심 AI 기능)
4. **설정 (`settings.html`)**
   - 디스코드/슬랙 웹훅 등록, 주간 리포트 수신 주기 설정

## 3. 핵심 기능

- 생각(실마리) 빠르게 던지기 → 즉시 저장
- 저장된 생각의 태그·시간 흐름을 시각적으로 확인 (타래장 그래프, 홈 태그 클라우드)
- 서로 다른 생각 2개를 AI로 조합해 새로운 기획 아이디어 도출 (실타래 연금술)
- 하나의 생각을 4가지 관점으로 확장해보는 베틀 기능
- 주간 활동 리포트를 디스코드/슬랙으로 자동 발송 (cron)
- 다크 모드 지원, 반응형 레이아웃(모바일/태블릿/데스크톱)

## 4. AI 기능 상세 — 실타래 연금술 (Idea Alchemy)

| 구분 | 내용 |
|---|---|
| 위치 | `vault.html` |
| 입력 | 타래장 그래프에서 사용자가 선택한 서로 다른 생각 노드 2개 (각각의 저장된 텍스트) |
| 처리 | `POST /api/expand` → Gemini(`gemini-2.5-flash`)에 두 생각을 함께 전달, "두 생각을 융합한 새로운 기획안"을 JSON으로 요청 |
| 출력 | 새로운 기획 제목 + 설명 텍스트를 결과 패널에 표시 |
| 사용자에게 제공하는 가치 | 서로 무관해 보이던 생각들 사이의 연결점을 AI가 대신 찾아주어, 혼자서는 떠올리기 어려운 새로운 기획 아이디어를 얻을 수 있음 |

### 실패 처리 기준

| 상황 | 처리 방식 |
|---|---|
| 노드 미선택(입력 없음) | 2개가 선택되기 전까지 조합 로직이 실행되지 않음 (버튼/트리거 비활성 상태 유지) |
| API 오류(4xx/5xx) | 오류 발생 시 조합을 재시도하지 않고, 화면에 안내 문구를 노출해 사용자가 상황을 인지하도록 처리 |
| 응답 지연 | 요청 중임을 알리는 로딩 문구("타래 내부 다중 전문가 패널을 소집하여 생각을 재조합하는 중...")를 결과 영역에 우선 표시 |

> ⚠️ **현재 구현 상태 메모**: 실제 코드에서는 `api/expand` 호출이 실패해도 사용자에게 별도 오류 메시지를 띄우지 않고 조용히 대체 텍스트로 넘어가도록 되어 있습니다. 평가 기준(실패 시 안내 메시지 노출)에 맞추려면 실패 시 명확한 안내 문구를 보여주는 방향으로 소폭 수정하는 것을 권장합니다. (별도로 안전하게 브랜치 분리해서 수정 가능)

## 5. 데이터 구조 (Supabase)

- `thoughts`: 사용자별 생각 파편 (content, tags, comfort, created_at, project_id)
- `projects`: 베틀에서 엮인 기획/할 일 목록 (title, roadmap, tasks, status)
- `daily_insights`: 일일 인사이트 캐시 (cron으로 매일 생성)
- `user_settings`: 웹훅 URL, 알림 채널 설정

## 6. 향후 확장 아이디어

- 홈 화면 입력 시 실시간으로 태그·공감 문구를 붙여주는 즉시 분류 기능 (백엔드 `classify.py`는 준비되어 있으나 프론트 연결은 아직)
- AI 기능 2개 이상으로 확장 시: 홈(즉시 분류) + 타래장(조합) + 베틀(확장)로 역할을 나눠 각기 다른 엔드포인트를 호출하도록 구조화 예정

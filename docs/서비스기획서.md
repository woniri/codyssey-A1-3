# 서비스 기획서 — 🧵 타래 (Tarae)

## 1. 서비스 개요

| 항목 | 내용 |
|---|---|
| 서비스명 | 타래 (Tarae) |
| 한 줄 정의 | 스쳐 지나가는 생각을 가볍게 기록하고, 서로 다른 생각을 엮어 새로운 아이디어로 발전시키는 개인 생각 관리 서비스 |
| 목적 | 메모 앱처럼 생각을 저장만 하는 데서 그치지 않고, 시간이 지나 잊혀진 생각들을 다시 연결해 실제 실행 가능한 기획으로 이어지도록 돕는다 |
| 타겟 사용자 | 사이드 프로젝트·콘텐츠 기획 등 여러 개의 아이디어를 동시에 굴리는 1인 개발자/기획자, 순간적인 생각이 자주 휘발되어 아쉬움을 느끼는 사용자 |
| 배포 URL | https://think-tarae.vercel.app/ |
| GitHub | https://github.com/woniri/codyssey-A1-3 |

## 2. 페이지/섹션 구성

내비게이션 바를 통해 4개 화면을 자유롭게 이동한다.

1. **실마리 (홈, `index.html`)**
   - 생각을 한 줄로 던지는 입력창 (빈 입력 시 안내 문구, 낙관적 UI로 즉시 응답감 제공)
   - **오늘의 실마리**: 매일 밤 배치로 생성되는 AI 인사이트를 실시간으로 불러와 표시
   - **머릿속 기상도**: 저장된 생각의 태그를 최대 15개, 등장 빈도에 따라 크기·색을 다르게 표시. 클릭하면 타래장에서 그 태그로 바로 필터링
   - **뜻밖의 공명**: 과거 생각 중 하나를 무작위로 소환해 보여주고, 클릭하면 타래장에서 해당 생각으로 포커스 이동
   - **하단 지표 3개**: 풀려 있는 실가닥(미분류)/단단해진 타래(프로젝트 수)/먼지 쌓이는 실가닥(14일 이상 방치)을 실시간 집계하고, 각각 클릭 시 베틀 또는 타래장(필터 적용)으로 이동
2. **베틀 (`loom.html`)**
   - 저장된 생각 목록에서 하나를 골라 AI 4인 전문가 패널(창의적 파괴자/비즈니스 아키텍트/제1원리 분석가/실타래 연결자)의 분석을 받아보는 작업대
3. **타래장 (`vault.html`)**
   - 전체 생각/프로젝트를 네트워크 그래프로 시각화
   - 태그별/프로젝트별/오래된 것만 필터링 가능 (홈 화면에서 딥링크로 들어오면 자동 적용)
   - 두 개의 생각 노드를 선택하면 AI가 조합하는 **실타래 연금술** 기능 제공 (핵심 AI 기능)
4. **설정 (`settings.html`)**
   - 자동발송 사용 여부, 사유 수확 범위(1일/1주/1달/직접입력), 발송 주기·시각 설정
   - 채널(디스코드/슬랙/노션/Gmail/텔레그램)을 **여러 개 동시에 등록**하고, 그중 발송에 포함할 채널을 개별 선택
   - 채널별 즉시 테스트 발송 기능

## 3. 핵심 기능

- 생각(실마리) 빠르게 던지기 → 즉시 저장 (낙관적 UI로 체감 속도 개선)
- 저장된 생각의 태그·시간 흐름을 시각적으로 확인 (타래장 그래프, 홈 태그 클라우드)
- AI가 생각 하나를 다각도로 분석 (베틀 - 4인 전문가 패널)
- AI가 서로 다른 생각 2개를 조합해 새로운 기획 아이디어 도출 (타래장 - 실타래 연금술)
- 여러 채널(디스코드/슬랙 등)을 동시에 등록해두고 원하는 채널로 리포트 테스트 발송
- 다크 모드 지원, 반응형 레이아웃(모바일/태블릿/데스크톱)

## 4. AI 기능 상세

### 4-1. 베틀 — 4인 전문가 분석 (Weave Tactics)

| 구분 | 내용 |
|---|---|
| 위치 | `loom.html` |
| 입력 | 사용자가 선택한 생각 1개(텍스트) |
| 처리 | `POST /api/expand` → Gemini(`gemini-2.5-flash`)에 "4명의 전문가 관점에서 분석해달라"는 프롬프트와 함께 전달, 지정된 JSON 포맷으로만 응답하도록 강제 |
| 출력 | 창의적 파괴자/비즈니스 아키텍트/제1원리 분석가/실타래 연결자 4개 카드에 각각의 분석 텍스트 표시 |
| 로딩 처리 | 요청 전 "⚡ 4대 사유 엔진 연동 중..." 문구를 먼저 렌더링한 뒤 `await`로 응답을 기다려, 사용자가 실제로 대기 상태를 인지할 수 있게 처리 |
| 실패 처리 | 요청 실패 시 "🌙 지금은 4대 전문가 패널을 소집하지 못했어요. 잠시 후 다시 직조해봐 주세요." 안내 문구를 표시 |

### 4-2. 타래장 — 실타래 연금술 (Idea Alchemy)

| 구분 | 내용 |
|---|---|
| 위치 | `vault.html` |
| 입력 | 타래장 그래프에서 사용자가 선택한 서로 다른 생각 노드 2개 |
| 처리 | `POST /api/connect` → Gemini에 두 생각(`text_a`, `text_b`)을 함께 전달, 병합된 새 아이디어(제목+설명)를 JSON으로 요청 |
| 출력 | 새로운 기획 제목 + 설명 텍스트를 결과 패널에 표시 |
| 로딩 처리 | "🚀 타래 내부 다중 전문가 패널을 소집하여 생각을 재조합하는 중..." 문구를 먼저 표시 |
| 실패 처리 | 요청 실패 시 "🌙 지금은 두 생각을 조합하지 못했어요. 잠시 후 다시 시도해봐 주세요." 안내 문구를 표시 |
| 사용자에게 제공하는 가치 | 서로 무관해 보이던 생각들 사이의 연결점을 AI가 대신 찾아주어, 혼자서는 떠올리기 어려운 새로운 기획 아이디어를 얻을 수 있음 |

두 기능 모두 정상 입력(2개 선택 완료) 상태에서만 실행되며, 서버 오류(4xx/5xx) 시 위와 같은 안내 문구로 사용자에게 상황을 알립니다.

## 5. 데이터 구조 (Supabase)

- `thoughts`: 사용자별 생각 파편 (content, tags, comfort, created_at, project_id)
- `projects`: 베틀에서 엮인 기획/할 일 목록 (title, roadmap, tasks, status)
- `daily_insights`: 일일 인사이트 캐시 (cron으로 매일 생성, 홈 화면이 실시간으로 조회)
- `user_settings`: 자동발송 사용 여부(`is_enabled`), 수확 범위(`harvest_scope`), 발송 주기(`send_frequency`), 발송 시각(`send_time`)
- `user_notification_channels`: 채널마다 한 행씩 저장(`channel`, `webhook_url`, `is_active`) — 여러 채널을 동시에 등록 가능. RLS 정책으로 본인 데이터만 접근 가능하도록 보호
